## 3. 임계값 상세 근거

### 3.1 ROLLING_WINDOW = 48개월

**근거**: settings.py 신청서 기준값. 48개월(4년)은 경기 순환 1주기를 포괄하여 전이율의 정상 분포를 안정적으로 추정할 수 있는 최소 기간이다.

**로버스트니스 체크**: W=36, 48, 60 세 가지를 동시 산출하여 비교한 결과, 구간별 탐지 건수 편차가 대부분 +-3건 이내로 안정적이었다. W=48 기본값 유지.

### 3.2 ZSCORE_ALERT = 2.5

**근거**: settings.py 신청서 기준값. 통계적 1% 수준 근사. ZSCORE_WARNING(2.0, 5% 수준)은 주의 레벨로 별도 기록하고, 최종 판정은 2.5 기준.

**데이터 검증**: 20개 구간(A, B)에서 warmup 제외 4,392건 중 182건(4.1%) 탐지. 구간당 연평균 약 1건으로, 과대/과소 탐지 없이 합리적 수준.

| Z-score 임계값 | 탐지 건수 | 비율 |
|---------------|----------|------|
| >= 2.0 | 278건 | 6.3% |
| >= 2.5 | 182건 | 4.1% |
| >= 3.0 | 127건 | 2.9% |
| >= 3.5 | 89건 | 2.0% |

### 3.3 IQR_MULTIPLIER = 1.5

**근거**: Tukey 표준(1977). 비모수적 이상치 판정 기준으로 널리 사용.

**데이터 검증**: Z-score 경보(>=2.5)와 IQR 이탈의 교차 판정 일치율이 100%. Z-score 경보 182건이 모두 IQR 이탈에도 해당. 모수적(Z-score)과 비모수적(IQR) 두 기준이 동일 시점을 가리켜, 탐지가 분포 가정에 민감하지 않음을 확인.

| | IQR 이탈 | IQR 정상 |
|---|---------|---------|
| Z-score 경보 | 182 | 0 |
| Z-score 정상 | 503 | 3,707 |

### 3.4 STABILITY_THRESHOLD = 3% (+-)

**근거**: settings.py 신청서 기준값. doc2_pattern_definitions_v2.md에서 "원화 환산 국제가의 월 변동률이 +-3% 이내인 기간"으로 정의.

**데이터 검증**: 10개 B구간에서 전체 관측 2,686건 중 안정 구간 1,291건(48.1%). 품목별 28%~62% 범위로, 분석에 충분한 안정 구간이 확보됨.

### 3.5 PATTERN3_N_VALUES = [2, 3, 6]

**근거**: settings.py 신청서 기준값. doc2에서 N=2(조기 신호), N=3(기본 탐지), N=6(구조적 이상)으로 3단계 동시 운용 명시.

**데이터 검증**: 단계적 필터링이 정상 작동.

| N값 | 역할 | 탐지 건수 | 축소율 |
|-----|------|----------|--------|
| N=2 | 조기 신호 | 271건 | - |
| N=3 | 기본 탐지 | 139건 | 51% (N=2 대비) |
| N=6 | 구조적 이상 | 29건 | 21% (N=3 대비) |

최종 판정(pattern3_flag)은 N=3 기본 탐지를 사용.

### 3.6 TRANSMISSION_RATE_MIN_UPSTREAM = 0.5%

**근거**: 상류 변화율이 0에 가까우면 전이율(하류/상류)이 폭발하여 Z-score를 왜곡한다. 이 임계값 미만이면 전이율을 NaN 처리하여 롤링 통계에서 제외.

**데이터 검증**:

| 임계값 | A/B 구간 제외 비율 | 전이율 절대최대 (밀 A 기준) |
|--------|-------------------|---------------------------|
| 0% (미적용) | 0% | 563 |
| 0.1% | 1.9% | - |
| 0.3% | 5.9% | - |
| **0.5%** | **10.1%** | **19.77** |
| 1.0% | 20.8% | - |

0.5%에서 전이율 극단값이 563 → 20 수준으로 억제되면서, A/B 구간 평균 10% 제외로 데이터 손실이 합리적. 패턴 2(전이율 Z-score) 미적용 구간(C, D, D')에서는 전이율 NaN 비율이 50~67%로 높지만, 이 구간은 패턴 2를 적용하지 않으므로 무관.

### 3.7 DIRECTION_REVERSAL_MIN_MAGNITUDE = 1.0%

**근거**: 가격 전달에 1~24개월 시차가 존재하므로, 같은 달에 상류와 하류가 반대 방향인 것은 자연스러운 현상이다. 크기 조건 없이 부호만 비교하면 전체 관측의 44%가 방향 역전으로 잡혀 이상 탐지로서 의미가 없어진다. 상류/하류 양쪽 절대값이 1.0% 이상일 때만 "유의미한 방향 역전"으로 판정한다.

**데이터 검증**:

| 임계값 | 방향 역전 건수 | 전체 관측 대비 |
|--------|-------------|---------------|
| 0% (미적용) | 3,897건 | 44.4% |
| 0.5% | 2,291건 | 26.1% |
| **1.0%** | **1,553건** | **17.7%** |
| 2.0% | 877건 | 10.0% |
| 3.0% | 491건 | 5.6% |

1.0%에서 17.7%로 축소. 시차 이탈 534건과 합산하여 패턴 1 총 탐지 2,007건(22.8%).

## 4. 설계 결정 상세 근거

### D1. 전이율 분모 처리 — NaN 처리 (A안)

**대안**: B안은 전이율을 0으로 대체. 그러나 "전달이 없었다"는 의미가 되어 실제 상황(판단 불가)과 부합하지 않음.

**채택 근거**: 상류가 거의 안 움직인 달에 전이율을 산출하는 것 자체가 무의미. NaN 처리하면 롤링 통계에서 자연스럽게 제외되어 Z-score 왜곡을 방지. 패턴 1 방향 역전 판정에는 원래 변화율(전이율 아님)을 사용하므로 영향 없음.

### D2. 구조 변화 시 기준선 — 하위 기간별 적용 (A안)

**대안**: B안은 전체 기간 기준선(baseline JSON의 normal_transmission_lag)을 일괄 적용.

**채택 근거**: Phase 6에서 구조 변화를 탐지하고 하위 기간별 모형을 재추정한 목적이 기간별 다른 기준선을 적용하기 위함. B안을 쓰면 Phase 6 산출물이 무의미해짐. 예: wheat D'에서 subperiod3(2020-11~)의 IRF 피크가 1개월인데 전체 기간 기준(21개월)을 쓰면 시차 이탈을 거의 탐지 불가.

**적용 구간**: 5개 구간(wheat D', maize D', palmoil D', sugar D', beef B). SubperiodResolver가 날짜를 subperiod에 매핑.

### D3. 롤링 윈도우 구조 변화 리셋 — 리셋 안 함 (A안)

**대안**: B안은 구조 변화 시점에서 롤링 윈도우를 초기화하고 새로 48개월 warmup.

**채택 근거**: 리셋하면 하위 기간별 탐지 가능 기간이 극히 짧아짐. 예: wheat D' subperiod3(2020-11~2026-02, 64개월)에서 48개월 warmup하면 탐지 가능 16개월뿐. 패턴 2는 A/B 구간만 적용되고, 구조 변화 5개 구간 중 A/B에 해당하는 건 beef B뿐이라 실질적 영향도 제한적.

**한계**: 구조 변화 이전 데이터가 롤링 윈도우에 포함될 수 있음. 논문에서 한계로 서술.

### D4. 시차 이탈 무반응 판정 — 같은 방향 반응 여부 (B안)

**대안**: A안은 하류 변화율 절대값이 임계값(예: 0.1%) 미만이면 무반응.

**채택 근거**: A안은 임계값 선택에 따라 결과가 크게 달라지고, 하류가 반대 방향으로 크게 움직인 경우를 잡지 못함(그건 방향 역전에서 별도 처리). B안은 (정상 시차 + 1)개월 윈도우 내에서 같은 방향 반응이 한 번이라도 있으면 "반응함"으로 처리하여, 추가 임계값 없이 판정 가능.

**데이터 검증**: 시차 이탈 534건의 92%가 정상 시차 1~2개월 구간에 집중. 정상 시차 24개월 구간에서는 0건으로 과탐 없음.

### D5. 스프레드 확대 판정 — 같은 부호 + 절대값 확대 (B안)

**대안**: A안은 부호 무관, |ECT_t| > |ECT_{t-1}|만 확인.

**채택 근거**: A안 vs B안 시뮬레이션 결과 N=3 기준 149건 vs 139건으로 차이 10건(7%)뿐. A안에서 추가로 잡히는 10건은 부호가 양→음 또는 음→양으로 급격히 뒤집히면서 절대값이 커지는 진동형 이탈. 패턴 3의 탐지 목적("한 방향으로 가격 괴리가 지속적으로 누적")에 부합하지 않음.

### D6. stat_timeseries 산출 — Phase 7에서 같이 생성 (A안)

**대안**: B안은 Phase 7은 패턴별 CSV만, stat_timeseries는 백엔드가 조합.

**채택 근거**: Phase 7에서 전이율, Z-score, IQR, ECT 등을 이미 전부 산출하므로 한 번에 출력하는 게 효율적. 백엔드가 산출 로직을 다시 구현할 필요 없이 CSV를 DB에 직접 적재.
